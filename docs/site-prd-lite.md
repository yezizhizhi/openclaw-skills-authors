# 首站 PRD（轻量版）

## 1. 核心用户是谁

### 核心用户

#### 1. 想用 AI 提高创作效率的内容创作者

- 典型身份：作者、自媒体写作者、内容运营、课程创作者、研究型写作者
- 真实状态：知道 AI 和 OpenClaw Skills 有用，但不知道先用哪个、怎么用
- 核心诉求：尽快找到适合当前任务的 Skill，而不是自己在海量列表里乱翻

#### 2. 已经在用 AI，但工具使用很分散的人

- 典型状态：试过很多工具、收藏了很多 Skill，但没有形成稳定工作流
- 核心诉求：按“写书 / 写文章 / 写文案 / 写报告 / 写论文 / 写课程”快速归类，减少试错

#### 3. 第一次接触 OpenClaw / ClawHub 的新手

- 典型状态：听说过 Skills，但还不理解它们适合什么场景
- 核心诉求：需要一个低门槛入口，先看懂“我该从哪里开始”

### 非核心用户

- 纯技术型开发者
- 只想看全部 Skill 原始列表的人
- 已经非常熟悉 ClawHub 并能自己高效检索的人

## 2. 页面要解决什么问题

### 核心问题 1：用户不知道自己该先看哪一类 Skill

首页需要先帮用户完成第一层判断：

- 我属于哪一类创作任务
- 我现在卡在什么工作环节
- 我应该先看哪一类 Skill

### 核心问题 2：用户在海量 Skill 里筛选成本太高

用户真正痛苦的不是“没有工具”，而是：

- 工具太多，不知道哪个好
- 很难判断哪个适合当前任务
- 经常点进无效链接或不对口页面

首页需要通过场景分类、精选推荐和 FAQ，把选择成本降下来。

### 核心问题 3：用户需要一个清晰的下一步动作

如果首页只讲概念，用户不会继续。
所以页面必须给出明确行动路径：

- 先按创作任务进入分类页
- 再按当前工作环节筛选
- 然后跳转到原始 Skill 页面继续使用

## 3. CTA 是什么

### 首页主 CTA

- `浏览全部 OpenClaw Skills`

作用：

- 让第一次来的用户快速进入场景分类
- 先建立“这个站是帮我找 Skill 的”认知

### 首页次 CTA

- `查看安装指南`

作用：

- 服务第一次接触 OpenClaw / Skill 安装流程的用户
- 降低新手理解门槛

### 场景型 CTA

- `点击进入skills库`

作用：

- 把用户从首页导向 6 个具体创作场景页
- 让用户更快进入与自己任务相关的内容

## 一句话总结

`clawauthor.com` 首页的核心任务，不是展示所有 Skill，而是帮助创作者先按任务和环节缩小范围，更快找到真正适合自己的 OpenClaw Skills。
